## 1. 注册开发者账号

### 1.1 注册

1. 打开 [华为开发者联盟](https://developer.huawei.com/consumer/cn/)
2. 点击右上角「注册」，使用华为账号登录
3. 选择开发者类型：
   - **个人开发者**：需要身份证实名认证
   - **企业开发者**：需要营业执照 + 法人信息

### 1.2 实名认证

1. 登录后进入 [开发者中心](https://developer.huawei.com/consumer/cn/console)
2. 点击「账号中心」→「实名认证」
3. 上传身份证正反面照片
4. 等待审核（通常 1-2 个工作日）

### 1.3 进入 AppGallery Connect

1. 打开 [AppGallery Connect](https://developer.huawei.com/consumer/cn/service/josp/agc/index.html)
2. 使用已认证的开发者账号登录

## 3. 申请签名证书

HarmonyOS 应用上架必须使用正式签名，流程分为三步：生成密钥 → 申请证书 → 申请 Profile。

### 3.1 生成密钥和 CSR 文件

在 DevEco Studio 中操作：

1. 菜单 → **Build** → **Generate Key and CSR**
2. **Key Store File**：选择存储路径，创建新的 `.p12` 文件
   - 设置密钥库密码（牢记此密码）
3. **Key Alias**：输入密钥别名，如 `cyberarcade`
   - 设置密钥密码
4. **CSR File**：选择 `.csr` 文件保存路径
5. 点击 **OK** 生成

生成后你会得到：
- `xxx.p12` — 密钥库文件（**绝对不要泄露和上传到 Git**）
- `xxx.csr` — 证书请求文件（用于上传到华为）

### 3.2 申请发布证书

1. AppGallery Connect → **用户与访问** → **证书管理** → **证书** 页签
2. 点击「新增证书」
3. 选择证书类型：**发布证书**
4. 上传刚才生成的 `.csr` 文件
5. 点击「提交」
6. 下载生成的 `.cer` 证书文件

### 3.3 申请 Profile 文件

1. AppGallery Connect → **用户与访问** → **证书管理** → **Profile** 页签
2. 点击「新增 Profile」
3. 填写信息：

| 字段 | 选择 |
|---|---|
| 类型 | 发布 |
| Bundle Name | `com.kerwin.cyberarcade` |
| 证书 | 选择上一步申请的发布证书 |
| 设备 | 不需要选择（发布 Profile 不限设备） |

4. 点击「提交」
5. 下载 `.p7b` Profile 文件

## 4. 配置签名到项目

### 4.1 DevEco Studio 配置

1. DevEco Studio → **File** → **Project Structure**
2. 左侧选择 **Signing Configs**
3. 取消勾选 **Automatically generate signature**
4. 手动填写：

| 字段 | 文件 |
|---|---|
| Store File | 选择 `.p12` 密钥库文件 |
| Store Password | 创建时设置的密钥库密码 |
| Key Alias | 创建时设置的别名（如 `cyberarcade`） |
| Key Password | 创建时设置的密钥密码 |
| Sign Alg | SHA256withECDSA |
| Profile File | 选择下载的 `.p7b` 文件 |
| Certpath File | 选择下载的 `.cer` 文件 |

5. 点击 **Apply** → **OK**

### 4.2 注意事项

- `.p12` 文件是私钥，**绝对不要**提交到 Git 仓库
- 确保 `.gitignore` 中包含 `*.p12`、`*.p7b`、`*.cer` 等签名文件
- 密码建议保存在安全的密码管理器中

在 AppGallery Connect 的应用信息页面，需要填写以下内容：

### 5.4 隐私政策

华为应用商店要求所有应用必须提供隐私政策链接。

**方案一：GitHub Pages 托管**

1. 在仓库中创建 `docs/privacy-policy.html`
2. 在仓库 Settings → Pages 中启用 GitHub Pages
3. 隐私政策 URL 为：`https://kerwin05.github.io/Game2048-OHOS/privacy-policy.html`

**方案二：其他方式**

- 使用免费的隐私政策托管平台
- 托管在个人网站上

**隐私政策参考内容**：


## 6. 构建 Release 包

### 6.1 构建步骤

1. 确保签名配置已完成（第 4 步）
2. DevEco Studio → **Build** → **Build Hap(s)/APP(s)** → **Build APP(s)**
3. 选择 **Release** 构建类型
4. 等待构建完成

### 6.2 输出文件

构建产物位于：

```
entry/build/default/outputs/default/
├── entry-default-signed.hap    # 签名后的 HAP 包
└── entry-default-signed.app    # 签名后的 APP 包（上架用）
```

上传到华为应用商店的是 **`.app`** 文件。

### 6.3 验证签名

构建完成后，确认输出文件名包含 `signed`，表示已正确签名。

## 7. 提交审核

### 7.1 上传安装包

1. AppGallery Connect → 你的应用 → **版本管理** → **准备提交**
2. 「软件版本」区域 → 点击「软件包管理」
3. 上传 `.app` 文件
4. 等待解析完成，确认包名和版本号正确

### 7.2 完善信息

确认以下信息已全部填写：

- [x] 应用图标
- [x] 应用截图（≥3 张）
- [x] 应用描述（简短 + 详细）
- [x] 隐私政策链接
- [x] 版权声明
- [x] 内容分级
- [x] 安装包已上传

### 7.3 提交

1. 确认所有必填项已完成（页面右侧有完成度提示）
2. 点击 **「提交审核」**
3. 确认提交

### 7.4 审核周期

- 通常 **1-3 个工作日**
- 审核结果会通过邮件和站内信通知
- 如果被驳回，根据驳回原因修改后重新提交

## 8. 常见问题

### Q: 审核被驳回怎么办？

常见驳回原因和解决方案：

| 驳回原因 | 解决方案 |
|---|---|
| 缺少隐私政策 | 补充隐私政策链接 |
| 截图不清晰 | 使用真机截图，确保分辨率达标 |
| 应用闪退 | 在目标设备上充分测试 |
| 功能不完整 | 确保所有功能可正常使用 |
| 描述与功能不符 | 检查描述文案是否准确 |

### Q: 需要软件著作权吗？

个人开发者上架休闲游戏通常不强制要求软著，但有软著审核会更快。

### Q: 可以免费上架吗？

是的。华为应用商店对开发者免费，不收取上架费用。

### Q: 上架后如何更新？

1. 修改 `app.json5` 中的 `versionCode`（递增）和 `versionName`
2. 重新构建 Release 包
3. 在 AppGallery Connect 创建新版本，上传新包
4. 提交审核

### Q: 签名文件丢失怎么办？

**签名文件一旦丢失无法恢复**，将无法更新应用。请务必：
- 备份 `.p12` 文件到安全位置
- 记录密钥库密码和密钥密码
- 不要将签名文件提交到 Git
